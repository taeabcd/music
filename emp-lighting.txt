STEP 1: CREATE EMPLOYEE OBJECT
Go to Setup
Click Object Manager
Click Create → Custom Object

Fill:

Label → Employee Validation
Object Name → Employee_Validation
Record Name → Employee Name
Data Type → Text

Click Save
-------------------------------------
Go to:
Employee Validation → Fields & Relationships → New
Create:
Employee ID
Type → Number
Label → Employee_ID

Salary
Type → Number
Label → Salary

Email
Type → Email
Label → Email

Department
Type → Picklist
Label → Department

Values:

HR
IT
Finance
Sales
Marketing

Joining Date
Type → Date
Label → Joining_Date

Go to:

Employee Validation → Validation Rules → New

1. Employee Name cannot be empty and minimum 3 characters

Rule Name:
Employee_Name_Validation
Formula:
OR(ISBLANK(Name), LEN(Name) < 3)
Error Message:
Employee Name cannot be empty and must contain at least 3 characters
Error Location:
Field: Employee Name
Click Save

2. Employee ID must be greater than 0
Rule Name:
Employee_ID_Validation
Formula:
Employee_ID__c <= 0
Error Message:
Employee ID must be greater than 0
Error Location:
Field: Employee ID
Click Save


3. Employee ID must be unique
This is not a validation rule.
Go to:
Employee Validation → Fields & Relationships → Employee_ID → Edit
Tick:
Unique
Click Save

4. Salary must be greater than 10,000 and less than 500,000
Rule Name:
Salary_Validation
Formula:
OR(Salary__c <= 10000, Salary__c >= 500000)
Error Message:
Salary must be greater than 10,000 and less than 500,000
Error Location:
Field: Salary
Click Save

5. Email must be valid
Since field type is Email, Salesforce automatically checks email format.
Still, create validation rule:
Rule Name:
Email_Validation
Formula:
NOT(REGEX(Email__c, "[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}"))
Error Message:
Please enter a valid email address
Error Location:
Field: Email
Click Save

6. Department must be selected
Rule Name:
Department_Validatio
Formula:
ISBLANK(TEXT(Department__c))
Error Message:
Please select a department
Error Location:
Field: Department
Click Save

7. Joining Date cannot be future date
Rule Name:
Joining_Date_Validation
Formula:
Joining_Date__c > TODAY()
Error Message:
Joining Date cannot be a future date
Error Location:
Field: Joining Date
Click Save
